Preferences

On this page you can set preferences for ExtendTube. There is no save/apply button and therefore preferences will take effect immediately and will affect all opened (YouTube) pages. However, there are some preferences that need to reload video player in order to be applied. They may reload video player or may not be applied until page is reloaded.

Help

This help is for version 1.21.

Interface

When ExtendTube is active you will see some buttons (if they are enabled) beneath video player (some of them are shown on figure 1). Their actions are explained below.

First button is Loop button. It has two functions. First is controlling section loop and second is controlling loop globally (it will change loop option in preferences). To enable or disable section loop just click on this button. If you want to toggle loop globally hold
Ctrl key down while clicking on it. The button will show what will happen when video riches end. Straight arrow means video will stop at end while circular arrow means it will loop. If icon is red that means that actual loop action is forced (this will happen if video is part of playlist or when section loop is enabled).
Note: Activation of loop and section loop was reversed prior to version 1.8.
Note 2: Since version 1.8.5 there are two new ways to toggle loop globally. First is middle click, and second click ’n’ hold.

When section loop is enabled two new buttons will appear (figure 3). These new buttons allows you to mark section of video you want to loop. You can click on them to set current playback progress as section start/end time or use mouse wheel to increment/decrement current time.

Second button (figure 4) from figure 1 is used for video downloading. When you click it a menu will appear with different download options.

In this menu you can choose quality of video you want to download. Available qualities are determined automatically and only those available are shown in menu (figure 5). See following table for details.

Last button on figure 1 is “preferences” button (figure 6). If you click on it it will open this page in which you can set preferences or read this help.

There are also three extra buttons (figure 7). By default they are not visible and you need to enable them in preferences. These buttons allows you to seek video back or forward or to play video one frame at a time. When enabled they will be placed before loop button.

Another button, which was introduced in version 1.8, is pop out button (figure 8). When you click on it, player will be taken out of normal content flow and centred on viewport with dark background around it.

In version 1.19.1 stop button was introduced (figure 9). When you click on it, the video will stop playing, if it's playing, and will stop buffering.

For some videos ExtendTube can load lyrics from an external source (lyrics.wikia.com). If lyrics are enabled ExtendTube will try to guess artist and title of the song from video title and it will show a button above video description (figure 10). Clicking this button will initiate lyrics search and, if found, lyrics will be shown. If ExtendTube cannot find artist and title no button will appear. However, you can still initiate lyrics search via keyboard shortcut (if you defined one) or you can force the button to show in all pages in preferences. If search fail you will be asked to enter artist and title of the song manually. Remember, lyrics are loaded from an external source and some may not exist on their servers.

Preferences

General section

Reset to default preferences!
When you click on default button all preferences will be cleared and YouTube will behave as ExtendTube is not installed. Only part of ExtendTube that will still work is hiding ads.
Load recommended preferences.
When you click on recommended button recommended preferences will be loaded. These are the preferences that are set when extension is run for the first time. This action will change only those preferences that are in recommended group; other will remain unchanged.
Export preferences!
If you want to make a backup of actual preferences click on export button. A new page will open with current preferences. Save that page so you can later import those preferences.
Import preferences
Opera extension cannot read local files, so, you have to paste preferences in import text-area. They must be in JSON format. You don’t have to enter whole preferences, just ones you actually want to change.

Extension section

Check for update
When this option is enabled ExtendTube will check for update every time it starts. That is on Opera start-up and when you manually disable and re-enable this extension.
Check for unapproved updates
The standard behaviour is check for updates from Opera extension catalog, because Opera needs to review new updates sent to their extension catalogue sometimes the update may take some time to be available. This option will let extension check for unapproved updates, updates in ExtendTube repository.
Update interval
This option will define the hour gap between update checks.
Allow auto update
This option will automatically download and install new updates. This only works for approved updates.
Add toolbar button
Add button to toolbar that can be used to control opened videos. This button will allow you to play or pause video when tab which plays video is not focused. It will also show how many videos (tabs) are currently opened. This number may not be correct if you go back/forward through tab history.
Add pop-up to toolbar button
When you click over the button the default behaviour is play/pause the latest opened video, this option will open a pop-up dialogue with a list of opened videos.
Always add pop-up
This option will always make the toolbar button open the pop-up dialogue, even if there’s only one open video. This options will override the above preference.
Pop-up update interval
This option define the update interval the extension will check for the status of opened videos. This option helps you tune your performance.
Log this type of messages
What type of messages should be logged to Opera’s Error Console (Ctrl+Shift+O).
nothing - don’t log anything
errors - log just errors
warnings - log errors and warnings
informations - log errors, warnings and informations
Replace invalid characters
When you choose to save video, video title will be used as file name. Invalid characters in video title will be replaced with this character(s). File name is then appended to download link and sent to YouTube as part of download request, so, YouTube may also remove or replace some characters.

Video player section

Hide in-video ads
Hide ads and video watermark inside flash player.
Buffer whole video
Videos when paused buffers only part ahead of its position to save bandwidth. This option will ignore this and will always buffer the whole video even when the video is paused. This option forces a feature removed and no more maintained by YouTube, like a compatibility mode, it's know to cause lot of bugs and/or unwanted behaviours like no 1080p or 144p options. If you have any problems with ExtendTube, try disabling this option.
Let extension write player
Hide ads and video watermark inside flash player.
Prevent cross play
This will prevent cross play around opened videos making only one video play at a time. Once a video is played any other video that is already playing will pause. This option is also respected with Play on focus preferences.
Disable autoplay
This will prevent videos from playing automatically at start, video will still buffer. Play on focus preference is respected.
Enable autoplay on playlists
Overwrite the disable autoplay option on playlists making the video autoplay on them.
Disable auto buffering
Prevents the video from downloading, it will only start downloading once you click, just like embedded videos.
Loop video
When this option is enabled video will play over and over again. However, this doesn’t work if video is part of a playlist because YouTube automatically advances to next video. If you want to prevent this behaviour you will have to enable sub-option.
Force loop
Loop will also be enabled if video is part of a playlist, preventing YouTube automatically advancing to next video.
Enable wide player
Enabling this option will make video player wider (expanded to 854x510 rather than default 640x390). Options are:
default - Youtube will decide when to use wide player
medium - standard wide size, as option on video player
large - larger video player for big screens
Force video quality
If set to other than default this option will force player to load video in chosen quality. it will also prevent player from changing quality when you expand player or go full screen. If chosen quality is not available first lower available quality will be loaded. Default means let player choose automatically.
Seek time
This is the time used when seeking video back and forward using extra buttons or via keyboard shortcuts. The forward time will always be little more than this time, and back time will be little less. This is because JavaScript code that controls player takes time to execute. For example: if you set this time to 5 seconds forward jump may be 6 and back 4 seconds.
Play on focus
If you want video to start playing when you focus tab enable this option. But, this option will be ignored if another video is already playing.
Force play on focus
Video will always play on focus, no matter if another video is already playing. Prevent cross play is respected and is recommended with this preference.
Only on first play
Video will only play on focus if playback is not started before.
Hide annotations
Hide overlay notes (annotations) inside video.
Note: When this option is changed flash player will not respond to change until page is reloaded.
Auto-hide controls
Auto-hide player controls. This option can be set to: disabled (controls and progress bar always visible), enabled (controls and progress bar will be hidden after few seconds), minimise progress (controls will be visible and progress bar will be minimised after few seconds) and show only progress (controls will be hidden and progress bar will be minimised after few seconds).
Note: When this option is changed flash player will not respond to change until page is reloaded.
Player theme
What theme video player should use: dark (black) or light (white).
Note: When this option is changed flash player will not respond to change until page is reloaded.
Progress colour
Choose colour for player’s progress bar (red or white).
Note: When this option is changed flash player will not respond to change until page is reloaded.

Interface section

Hide ads on page
Hide ads on YouTube pages. Not on videos.
Don’t detect language
By default ExtendTube will detect language that YouTube currently uses and will use the same language for content that adds to page. If you enable this option, ExtendTube will always use language that you can choose upon enabling this option.
Note: This option will change language only for content added by this extension not the whole page.
Enable download
This option will add button that, when activated, will show menu with download options for playing video. When this option is enabled a new option will be available.
Use fallback servers
This option allow you to use alternate servers for download links.
Add stop button
This option will add button that can be used to stop video playback and buffering. This button helps everybody that miss the right-click Stop download option.
Add loop button
This option will add button that can be used to enable or disable Loop video option. This option let you define the start and end positions of the video to loop.
Add pop out player button
Adds button that can be used to pop player out of page and centre it on the viewport.
Resize player on pop out
When it’s enabled you can choose whether you want player to be resized to display real video size. If you choose for player to be resized it will be expanded or shrunk to size of video or viewport size if video is greater than viewport. Otherwise, player size will be 854x510 pixels.
Add extra buttons

Keyboard shortcuts

Enable keyboard shortcuts
Set this option to enable keyboard shortcuts.
YouTube flash player have some keyboard shortcuts predefined, but they work only if player is focused. If you enable Expose player shortcuts sub-option this extension will try to make them work even when player is not focused.
Note: If some action (ex. play/pause) is executed twice when you interact (click) with flash player and player shortcuts are exposed to page please enable “Disable flash player shortcuts” sub-option.

When defining shortcuts it is recommended, but not absolutely necessary, to use modifier keys(?). Note that some key combination are reserved by system and should be avoided (like Alt+F4 or
Alt+f). Also, when some elements on page are focused (like search field or a button) some keyboard shortcuts may not work.

If you enabled “expose player shortcuts” option you will have the following shortcuts available. They cannot be redefined but they will be executed only if no other action is defined for shortcut.

  • 0, 1,... 9 - jump to 0%, 10%,... 90% of video
  • Home - jump to beginning of video
  • End - jump to end of video
  • Left - jump 10% back
  • Right - jump 10% forward
  • Up - increase volume 5%
  • Down - decrease volume 5%
  • Space - play/pause video

Note: since version 1.8 you cannot set Esc as keyboard shortcut (unless you do a little hacking, but it will not work even if you do).

Actions

Actions are JavaScript snippets that can be executed by this extension. They are executed by being passed to function constructor as an argument. 

Function("xtt", "<action_code>")(window.opera.xtt);

This is equivalent to: 

// anonymous function executed in global scope
(function (xtt) {
	// action code goes here
})(window.opera.xtt);

So, everything that is created by an action will be destroyed after action is executed. If you want something to be available after action is done executing, make it property of some globally accessible object. 

var run_counter = 0;
if (window.run_counter !== undefined)
	run_counter = window.run_counter;
run_counter++;

console.log("This action is run " + run_counter + " times.");
window.run_counter = run_counter;

Adding a new action

When adding a new action you need to define three things. First you need to define unique action ID. Second, you need to define when action should be executed and last, you need to enter action code. Then just click Save action button.

Actions are executed when some event is triggered. If some event is fired before actions are sent by background process, actions defined for that event will be executed immediately after they are received. Events that can trigger an action are:

immediately
Actions are executed immediately (after some initialisation).
DOM created
Actions are executed when DOMContentLoaded event is fired.
page loaded
Actions are executed when load event is fired.
document modified
Actions are executed after extension is done modifying document (after buttons are added).
plug-in initialised
Actions are executed when flash plug-in is initialised.
Note: This event may be fired several times (if flash player is replaced for some reason).
player ready
This event will be triggered twice. First time when video player becomes ready to play video, at the same time background process will be signaled about player being ready. And second time when background process returns message.
Note: This event may be fired several times (if flash player is replaced for some reason).
playback ended
Actions are executed when video playback reaches end.
playback started
Actions are executed when video starts playing (whether for the first time or after being paused).
playback paused
Actions are executed when video playback is paused.
develop
This is a special event. Actions with this event will be executed every time preferences are received from background process which is every time you save an action.

As you may guess some events will fire only if page contains a video.

If you are writing a new action it is recommended to use “develop” event. First open a YouTube page, then write action, click Save action button and watch Error Console (Ctrl+Shift+O) for errors. If there are errors, fix them and click Save again and action will be executed once again. There is no need to reload YouTube page. If everything went without errors, choose different event trigger and you are done.

While writing action code you may call some methods of extension object. ExtendTube object is property of Opera object. See API for details. 

// reference to ExtendTube object.
window.opera.xtt;

Action examples 

// This action will prevent video auto buffering.
// It should be executed on event “player ready”.

// Give time to flash player to finish initialising.
window.setTimeout(function () {
	xtt.player.control("stop");
}, 1000);
// If you have fast machine you can reduce time out delay.
// This action will add option to download video as mp3.
// It should be executed on event “document modified”.

xtt.ui.add.downloadMenuItem({
	text: "mp3 download HQ",
	tooltip: "Download as high quality mp3",
	uri: "http://www.video2mp3.net/?hq=1&url=" + xtt.video.getVideoURI(true)
});

// This action will open conversion window in new tab.
xtt.ui.add.downloadMenuItem({
	text: "mp3 download",
	tooltip: "Download as standard quality mp3",
	uri: "http://www.video2mp3.net/?url=" + xtt.video.getVideoURI(true),
	clickListener: function (event) {
		event.preventDefault();
		window.open(this.href);
    }
});

Extension pop-up

Depending on your settings, when you click on extensions button a pop-up may appear. In this pop-up you’ll see a list of all opened videos. Every list entry include the video title, current video progress and playback control button.

For embedded videos, clicking on video title will open video in a new regular YouTube page. If video is on a regular YouTube page, tab will be focused. In order for tab focus to work you have to enable Allow raising of windows option in Preferences/Advanced/Content/JavaScript Options... .

In front of video title an icon may appear showing some info about video:

  • - video is a HTML5 video;
  • - video is an iframe embedded video.

Progress bar will show current video progress and it will allow you to seek the video. In some versions of Opera it will change colour depending of whether playback is paused (yellow) or playing (green).

Control button will show current playback state ( - paused; - playing) and it will allow you to change that state by clicking on it.

Pop-up will use Custom colours if you’ve enabled them.
